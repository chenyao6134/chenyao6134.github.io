AegisMark Privacy Policy

Last Updated: June 13, 2026

Scope

This Privacy Policy applies only to AegisMark, a desktop watermarking application published by Massimo Yao ("Developer").

AegisMark is designed to perform its core features locally on the user's device. The app does not upload watermark templates, screen contents, browser URLs, screenshot files, or device information to developer servers.

Data Processed by AegisMark

AegisMark may process the following data locally to provide watermarking features:

  • Watermark layers, styles, layout settings, display rules, custom variables, keyboard shortcuts, and app preferences.
  • User-selected watermark images and user-selected screenshot folders.
  • Local device context used in watermark variables, such as user name, computer name, device model, date, time, display name, frontmost app name, and frontmost app bundle identifier.
  • Browser URL or website domain information when the user enables website-based watermark rules and grants Automation permission.
  • Network SSID information when the user enables network-based watermark rules.

This data is processed locally and is not transmitted to the Developer.

Screen Recording Permission

AegisMark uses macOS Screen Recording permission only for window masking. Window masking allows the app to hide watermark layers inside user-selected app window regions while keeping the watermark visible elsewhere.

When this feature is enabled, AegisMark reads on-screen window metadata such as window bounds, window number, owning process ID, and the associated app bundle identifier. This metadata is used only to calculate the regions where watermark layers should be hidden.

AegisMark does not use Screen Recording permission to record video, capture audio, perform OCR, upload screen contents, or send screen data to developer servers.

Screen Recording window metadata is processed in memory and is not stored by the Developer.

Screenshot Watermarking

Screenshot watermarking is performed on image files in a folder selected by the user. AegisMark monitors the selected folder, reads recognized screenshot files locally, applies visible watermark layers, and saves a watermarked output file to the same selected folder.

If invisible watermarking is available, AegisMark may embed a local payload into the output image. The payload may include user name, computer name, device model, source file name, timestamp, active app name, and active app bundle identifier.

The Developer does not receive screenshot files, screenshot contents, or embedded watermark payloads. Watermarked screenshots remain in the user-selected folder until the user deletes or moves them.

Automation Permission

AegisMark may request Automation permission to read the active browser URL for website-based watermark rules. This is used to decide whether a user-configured watermark rule should be active for the current website.

Browser URL and website domain information are processed locally and are not transmitted to the Developer.

Files and Folders Access

AegisMark uses user-selected file and folder access to read watermark images and process screenshots in a user-selected folder. The app stores security-scoped bookmarks locally so it can continue to access the selected folder after the user enables screenshot watermarking.

Only files and folders selected or authorized by the user are accessible to the app.

Local Storage and Retention

AegisMark stores app settings, watermark configurations, rules, custom variables, keyboard shortcuts, screenshot folder security bookmarks, and purchase status in the app sandbox on the user's device.

This local data is retained until the user changes the settings, removes generated files, resets the app, or deletes the app and its associated data.

Purchases

AegisMark uses Apple StoreKit for in-app purchases, subscriptions, purchase restoration, and lifetime unlocks. Apple manages the payment flow. The Developer does not directly receive payment card information.

Third-Party Sharing

AegisMark does not sell user data, use user data for advertising tracking, or share screen recording data, screenshot contents, browser URLs, watermark settings, or local device context with third parties.

Information may be disclosed only if required by applicable law, regulation, legal process, or governmental request.

Children's Privacy

AegisMark is not directed to children under the age of 13. The Developer does not knowingly collect personal information from children.
